<- four mocks audit / data proposal

design follows the contract

The pane has a hierarchy problem.

The current slips repeat where, model, and duration, but they flatten current state, project context, stream health, and meaningful outcomes. Better layout starts by preserving better data.

Current gaps

  1. Only two live agents render even when the API returns more, and unsorted insertion order decides which two.
  2. Blocked agents stay open in the read model but appear as working in the interface.
  3. Six-hour merging can erase individual run boundaries and duration distribution.
  4. At-least-once delivery has no stable event ID or ingest idempotency, so totals can inflate.
  5. Snapshots are timestamped but never used to reconcile stale or missing live state.
  6. Host, focus intervals, cache counters, collector health, and freshness are stored or returned but discarded.
  7. API failure can look like an empty desk while the footer still claims the stream is live.

Privacy line

Prompts remain private. The collector should never publish prompt or response text, pane content, transcript structure, terminal previews, tool names, file paths, or OSC titles. A private workspace can still report presence, state, and coarse timing while its identity and work remain redacted.

OpenTelemetry's current GenAI registry is a useful naming reference for agent, provider, model, workflow, and token fields. It also warns that message attributes can contain sensitive information, which supports keeping this stream counters-only.
